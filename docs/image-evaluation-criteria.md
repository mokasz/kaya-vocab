# 単語イラスト 評価基準

対象: Kaya's Vocab フラッシュカード用 AI生成イラスト（Imagen 4.0）
目的: 中学1年生（12歳女子）が英単語を視覚的に記憶できるか

---

## 評価軸（5項目）

### 1. 関連性 (Relevance)
**問い:** イラストだけを見て、その英単語の意味が伝わるか？

| スコア | 基準 |
|--------|------|
| ✅ OK | 単語の意味が一目で分かる |
| ⚠️ Keep | 意味は伝わるが、やや間接的 |
| ❌ NG | 単語と無関係、または別の意味に見える |

### 2. 明確性 (Clarity)
**問い:** 主役の物・人がはっきり描かれているか？

| スコア | 基準 |
|--------|------|
| ✅ OK | 主役が明確、背景がシンプル（白/薄色） |
| ⚠️ Keep | 主役は分かるが背景がやや煩雑 |
| ❌ NG | 何が描かれているか分かりにくい、または小さすぎる |

### 3. 年齢適切性 (Age-appropriateness)
**問い:** 12歳女子向けとして適切か？

| スコア | 基準 |
|--------|------|
| ✅ OK | 明るく親しみやすい、怖くない |
| ⚠️ Keep | 問題はないが、対象年齢より幼い/大人向け |
| ❌ NG | 暴力的・怖い・不適切な表現を含む |

### 4. 記憶補助力 (Memorability)
**問い:** このイラストが単語の記憶のフックになるか？

| スコア | 基準 |
|--------|------|
| ✅ OK | 見たらすぐ単語が思い出せそう（強い視覚的連想） |
| ⚠️ Keep | 覚えるのに使えるが、インパクトが弱い |
| ❌ NG | 抽象的すぎて記憶の助けにならない |

### 5. 視覚品質 (Visual Quality)
**問い:** イラストとして品質が十分か？

| スコア | 基準 |
|--------|------|
| ✅ OK | 鮮明、適切なサイズ感、白背景 |
| ⚠️ Keep | 若干ぼやけているが許容範囲 |
| ❌ NG | ぼやけている、文字が混入している、生成失敗 |

---

## 総合判定ルール

- 全項目 ✅/⚠️ → **PASS**（そのまま使用）
- ❌ が1つでも → **FAIL**（プロンプト改善して再生成）

## 評価ワークフロー（自動化版）

`generate_images.py` が Gemini → 自動評価 → Vertex AI フォールバックを自動実行する。

```
venv/bin/python kaya-vocab/scripts/generate_images.py [--force] [--word <id>]

各単語:
  Gemini API で生成（無料）
       ↓
  Gemini Vision で自動評価（3軸: text/relevance/quality）
       ↓ PASS → data/images/{id}.png に保存
       ↓ FAIL（テキスト混入 / 意味が曖昧 / 品質不良）
  Vertex AI + negative_prompt で再生成
       ↓
  再評価 → 保存（FAIL でも保存し ⚠️ 警告表示）
```

### API 選択の設計方針

| API | 用途 | 費用 |
|-----|------|------|
| Gemini API | 第1候補（無料） | 無料枠 |
| Vertex AI | FAILフォールバック | ~$0.02/枚 |

- **`negative_prompt`** は Vertex AI のみ対応（Gemini API 非サポート）
- **`PROMPT_OVERRIDES`** dict で単語名が看板になりやすい語を場面描写で上書き
  - 例: `store` → 「母と子がスーパーの棚を歩く」場面に置き換え

### 手動確認フロー（オプション）

```
1. python3 -m http.server 8080 --directory kaya-vocab
2. ブラウザコンソールでギャラリー表示スクリプトを実行
3. 目視で確認 → 問題があれば PROMPT_OVERRIDES に追加して --force 再生成
4. ユーザーの最終承認 → デプロイ
```
